# 蓝鲸CRM AI 交互模式方案

## 1. 目标

在不影响现有菜单、表单、列表页交互的前提下，为 CRM 增加一种面向人工智能的自然语言交互模式。用户可以直接输入业务诉求，系统返回对应的界面结果、摘要结论和可执行操作。

核心目标：

- 保留现有交互不变，新增并行入口
- 让用户可以通过自然语言完成查询、汇总、定位和草稿生成
- 将 AI 输出尽量落回现有页面和现有 API，减少重复建设
- 在早期阶段严格限制风险，避免 AI 直接越权或误写数据

## 2. 产品形态建议

推荐形态：`智能助手` 页面 + 全局右侧抽屉

建议入口：

- 顶部导航新增 `智能助手`
- 关键列表页右上角新增 `AI 帮我找`
- 客户详情页增加 `AI 生成下一步建议`

建议交互方式：

1. 用户输入自然语言，例如：
   - “帮我找最近 7 天没有跟进的客户”
   - “查看王越名下金额大于 10 万的商机”
   - “总结最近成交机会最多的销售成员”
   - “给这个客户生成下一步跟进建议”
2. AI 解析成结构化意图与参数
3. 系统调用受控业务工具或现有 API
4. 结果以两种形式返回：
   - 摘要卡片
   - 跳转到现有页面并自动带上筛选条件

## 3. 第一阶段能力边界

第一阶段只做“查询和建议”，不做直接写库。

### 3.1 支持的高频意图

- 查客户
- 查待跟进客户
- 查商机
- 查订单
- 查销售/客户摘要
- 查某客户最近互动
- 生成下一步跟进建议

### 3.2 暂不开放

- AI 直接创建客户
- AI 直接新增订单
- AI 直接修改商机阶段
- AI 自由生成 SQL
- AI 绕过权限直接读取全量数据

## 4. 系统架构建议

建议采用“受控编排”架构，而不是让模型直接访问数据库。

### 4.1 前端层

新增模块：

- `AI Assistant` 页面
- `AssistantDrawer` 全局抽屉
- `ResultCards` 结果卡片组件
- `ActionPreviewCard` 操作预览卡片

前端职责：

- 收集自然语言输入
- 展示 AI 返回的摘要、列表和建议
- 将结构化条件同步到现有页面路由参数
- 对需要确认的操作显示确认卡片

### 4.2 后端层

新增统一入口：

- `POST /api/ai/assist`

建议拆分三层：

1. `intent resolver`
   - 判断用户是在查询、汇总、解释还是准备执行操作
2. `tool router`
   - 将结构化意图映射到受控业务工具
3. `response formatter`
   - 返回摘要、列表、页面跳转参数和后续建议

### 4.3 AI/工具层

模型不直接查数据库，只能调用白名单工具。

建议第一版工具：

- `search_customers`
- `search_opportunities`
- `search_orders`
- `get_dashboard_summary`
- `get_customer_interactions`
- `generate_follow_up_suggestion`

后续带确认写操作时再增加：

- `create_interaction_draft`
- `create_customer_draft`
- `update_customer_status_draft`

## 5. API 设计草案

### 5.1 请求

`POST /api/ai/assist`

```json
{
  "message": "帮我找最近 7 天没有跟进的华东客户",
  "context": {
    "page": "/customers",
    "customer_id": null
  }
}
```

### 5.2 响应

```json
{
  "success": true,
  "data": {
    "intent": "search_customers",
    "summary": "找到 12 位最近 7 天未跟进的华东客户。",
    "filters": {
      "region": "华东",
      "last_interaction_before_days": 7
    },
    "result_mode": "page_and_summary",
    "target_page": "/customers",
    "target_query": {
      "stale_days": 7,
      "region": "华东"
    },
    "cards": [
      {
        "type": "metric",
        "title": "待跟进客户",
        "value": 12
      }
    ],
    "items": []
  }
}
```

### 5.3 带确认写操作响应

```json
{
  "success": true,
  "data": {
    "intent": "create_interaction_draft",
    "summary": "已生成一条电话跟进草稿，请确认后写入系统。",
    "result_mode": "draft_confirmation",
    "draft_action": {
      "type": "create_interaction",
      "customer_id": 18,
      "payload": {
        "interaction_type": "call",
        "subject": "AI 生成电话跟进草稿",
        "description": "客户关注报价进度，希望下周收到更新。",
        "outcome": "neutral",
        "next_action": "下周发送更新版报价"
      }
    }
  }
}
```

## 6. 前端页面原型建议

### 6.1 智能助手页面

布局建议：

- 左侧：对话区
- 右侧：结果卡片区
- 顶部：快捷问题建议
- 底部：输入框和发送按钮

核心区域：

- `最近问题`
- `快捷提问`
- `AI 结果摘要`
- `跳转到客户页/销售页/报表页`
- `待确认操作`

### 6.2 页面联动方式

尽量复用现有页面：

- 客户查询结果：跳 `/customers`
- 商机查询结果：跳 `/sales`
- 报表摘要：跳 `/reports`
- 某客户最近互动：跳 `/customers?customer_id=...`

### 6.3 结果卡片类型

- `summary`
- `metric`
- `list`
- `warning`
- `next_actions`
- `draft_confirmation`

## 7. 权限与安全约束

必须遵守以下规则：

1. AI 只能访问当前用户已有权限的数据范围
2. 第一阶段 AI 不直接执行写操作
3. 第二阶段所有写操作必须经过用户确认
4. 所有 AI 触发的操作都记录审计日志
5. 模型只能调用白名单业务工具，不能自由拼接 SQL

## 8. 分阶段实施计划

### Phase 1：查询与摘要

目标：

- 自然语言查询客户/商机/订单
- 返回摘要与页面跳转结果
- 支持客户最近互动和报表摘要

交付项：

- 后端 `/api/ai/assist`
- 5 到 6 个只读工具
- 前端 `智能助手` 页面
- 基本提示词与结构化协议

验收标准：

- 至少 20 条中文业务指令可被稳定解析
- 结果可以正确跳转到现有页面
- 不影响现有交互和权限体系

### Phase 2：带确认的草稿操作

目标：

- AI 生成跟进草稿
- AI 生成客户草稿
- AI 生成商机草稿

交付项：

- `draft_confirmation` 结果类型
- 确认执行 API
- AI 操作审计日志

验收标准：

- 用户确认前不落库
- 确认后可调用现有业务接口完成写入

### Phase 3：工作流助手

目标：

- 待跟进任务整理
- 跟进建议生成
- 销售摘要和优先级建议

交付项：

- 任务建议卡片
- 推荐动作卡片
- 上下文连续对话能力

## 9. 开发任务拆分

### 9.1 后端任务

- 新增 `routes/ai.py`
- 新增 `services/ai_assistant.py`
- 定义工具协议与白名单工具
- 接入权限检查与数据范围控制
- 增加 AI 审计日志模型或日志文件

### 9.2 前端任务

- 新增 `pages/AIAssistant.tsx`
- 新增 `components/assistant/*`
- 新增路由与导航入口
- 增加结果卡片渲染器
- 增加页面跳转参数同步

### 9.3 测试任务

- 意图解析测试
- API 集成测试
- 页面级回归测试
- 权限隔离测试
- 带确认操作链路测试

## 10. 推荐起步顺序

建议严格按下面顺序推进：

1. 先确定 20 到 30 条真实中文业务语料
2. 再定义结构化响应协议
3. 然后实现只读 AI 编排接口
4. 再做前端 `智能助手` 页面
5. 最后再开放带确认的草稿写操作

## 11. 当前建议的第一批示例语料

- 帮我找最近 7 天没跟进的客户
- 查一下王越名下所有商机
- 看一下最近新增订单最多的销售
- 这个客户最近一次互动是什么
- 总结一下当前销售漏斗
- 哪些客户已经进入方案报价阶段
- 给这个客户生成下一步跟进建议
- 列出本周新增的高价值线索

## 12. 结论

最推荐的落地方式是：

- 保持现有 CRM 页面不变
- 新增一个 `智能助手` 作为第二交互层
- 第一阶段只做查询和建议
- 第二阶段再做带确认的写操作

这样既能快速体现自然语言交互价值，也能把风险控制在现有业务体系之内。
